# gpu-compressor CLI reference

## Overview

gpu-compressor compresses and decompresses files using NVIDIA GPU acceleration. Two algorithms are supported: Zstd (`.nvzs`) and LZMA2 (`.nvlz`). Format is auto-detected on decompression.

```
gpu-compressor <COMMAND> [OPTIONS]

Commands:
  compress    Compress one or more files
  decompress  Decompress one or more compressed files
```

---

## compress

Compress one or more input files or a directory.

```
gpu-compressor compress -i <INPUT>... [-o OUTPUT] [-a ALGORITHM] [-d DEVICE] [-l LEVEL] [--chunk-size BYTES] [--dict-size MB]
```

### Required

| Flag | Description |
|------|-------------|
| `-i, --input <INPUT>...` | Input file(s) or a single directory. Accepts one or more paths. |

### Optional

| Flag | Default | Description |
|------|---------|-------------|
| `-o, --output <OUTPUT>` | auto | Output path. See output naming rules below. |
| `-a, --algorithm <ALGORITHM>` | `zstd` | Compression algorithm. `zstd` or `lzma2`. |
| `-d, --device <DEVICE>` | `0` | CUDA device ID (0-indexed). |
| `-l, --level <LEVEL>` | `0` | Compression level. See level table below. |
| `--chunk-size <BYTES>` | `134217728` | Chunk size in bytes for multi-file Zstd L0. Default 128 MiB. |
| `--dict-size <MB>` | `128` | LZMA2 dictionary size in MiB. Sets pipeline chunk size. Dictionary stored in host RAM, not VRAM. |

### Compression levels

| Algorithm | Level | Match Finding | Entropy Coding | Notes |
|-----------|-------|---------------|----------------|-------|
| zstd | 0 | nvCOMP internal | nvCOMP internal | Fastest. Dual GPU auto-enabled. |
| zstd | 1 | GPU LZ77, depth 16 | CPU ZSTD_compressSequences | Better ratio. Single GPU only. |
| zstd | 2 | GPU LZ77, depth 64 | CPU ZSTD_compressSequences | Best ratio. Single GPU only. |
| lzma2 | 0 | liblzma internal | liblzma range coder | 8-worker multi-threaded. Preset 0-9 via `-l`. |
| lzma2 | 1+ | GPU HC4, depth 32 | liblzma preset 9 (N CPU threads) | Async GPU→buffer→CPU pipeline. Both GPUs if available. |

For LZMA2 L0, `-l` maps to liblzma preset (0 = fastest, 9 = best ratio). For LZMA2 L1+, `-l` selects the GPU+CPU async pipeline regardless of value.

### Output naming

- **Single file, no `-o`:** output placed alongside input with `.nvzs` or `.nvlz` appended.
  - `data.bin` → `data.bin.nvzs`
- **Single file, `-o` is a file path:** use that path directly.
- **Single file, `-o` is a directory:** output placed inside that directory with auto-generated name.
- **Multiple files, no `-o`:** each output auto-generated alongside its input.
- **Multiple files, `-o` is a directory:** all outputs placed inside that directory.
- **Multiple files, `-o` is a file:** `-o` is ignored; all outputs auto-generated.
- **Directory input:** all files recursively discovered; outputs placed under the specified output directory (or input directory if `-o` omitted) preserving relative structure.

### Directory compression

When `-i` is a single directory, all files within are recursively compressed individually. Directory structure is preserved under the output path. LZMA2 directory compression is not supported.

### Dual GPU

Zstd L0: auto-detects GPU count; 2+ GPUs → even/odd chunk interleaving across GPU 0 and GPU 1.

LZMA2 L1+: both GPUs run match finding concurrently, feeding the shared CPU encoder pool via MPMC channel.

LZMA2 L0 and Zstd L1/L2: single GPU only.

To force single GPU:
```bash
CUDA_VISIBLE_DEVICES=0 gpu-compressor compress -i file.bin
```

## decompress

Decompress one or more compressed files. Algorithm and format are auto-detected from the file magic (`NVZS` or `NVLZ`).

```
gpu-compressor decompress -i <INPUT>... [-o OUTPUT] [-d DEVICE]
```

### Required

| Flag | Description |
|------|-------------|
| `-i, --input <INPUT>...` | Compressed file(s). Accepts one or more paths. |

### Optional

| Flag | Default | Description |
|------|---------|-------------|
| `-o, --output <OUTPUT>` | auto | Output path. See output naming rules below. |
| `-d, --device <DEVICE>` | `0` | CUDA device ID (used for Zstd decompression via nvCOMP). |

### Output naming

- **Single file, no `-o`:** strip compression extension.
  - `data.bin.nvzs` → `data.bin`
  - `data.bin.nvlz` → `data.bin`
  - Unknown extension: append `.out`
- **Single file, `-o` is a file path:** use that path directly.
- **Single file, `-o` is a directory:** output placed inside that directory with auto-generated name.
- **Multiple files, no `-o`:** each output auto-generated by stripping extension.
- **Multiple files, `-o` is a directory:** all outputs placed inside that directory.
- **Multiple files, `-o` is a file:** `-o` is ignored; all outputs auto-generated.

## Examples

### Compress a single file (Zstd, default)

```bash
gpu-compressor compress -i data.bin
# → data.bin.nvzs
```

### Compress with explicit output path

```bash
gpu-compressor compress -i data.bin -o /backups/data.nvzs
```

### Compress with Zstd level 1 (GPU match finding + CPU FSE)

```bash
gpu-compressor compress -i data.bin -l 1
```

### Compress with Zstd level 2 (optimal matching)

```bash
gpu-compressor compress -i data.bin -l 2
```

### Compress with LZMA2 (liblzma, 8 workers)

```bash
gpu-compressor compress -i data.bin -a lzma2
# → data.bin.nvlz
```

### LZMA2 with custom preset and dictionary size

```bash
gpu-compressor compress -i data.bin -a lzma2 -l 6 --dict-size 256
```

### LZMA2 L1 (async GPU+CPU pipeline)

```bash
gpu-compressor compress -i data.bin -a lzma2 -l 1
```

### Compress multiple files

```bash
gpu-compressor compress -i file1.bin file2.bin file3.bin
# → file1.bin.nvzs  file2.bin.nvzs  file3.bin.nvzs
```

### Compress multiple files to a directory

```bash
gpu-compressor compress -i file1.bin file2.bin -o /backups/
# → /backups/file1.bin.nvzs  /backups/file2.bin.nvzs
```

### Compress a directory recursively

```bash
gpu-compressor compress -i /data/models/ -o /compressed/models/
```

### Select a specific GPU

```bash
gpu-compressor compress -i data.bin -d 1
```

### Force single GPU via environment

```bash
CUDA_VISIBLE_DEVICES=0 gpu-compressor compress -i data.bin
```

### Decompress (auto-detects NVZS or NVLZ)

```bash
gpu-compressor decompress -i data.bin.nvzs
# → data.bin

gpu-compressor decompress -i data.bin.nvlz
# → data.bin
```

### Decompress with explicit output

```bash
gpu-compressor decompress -i data.bin.nvzs -o /restored/data.bin
```

### Decompress multiple files

```bash
gpu-compressor decompress -i a.nvzs b.nvzs c.nvlz
```

### Decompress to a directory

```bash
gpu-compressor decompress -i a.nvzs b.nvzs -o /restored/
```

## File Extensions

| Extension | Format | Magic | Decompressor |
|-----------|--------|-------|--------------|
| `.nvzs` | Zstd compressed | `NVZS` | nvCOMP `nvcompBatchedZstdDecompressAsync` |
| `.nvlz` | LZMA2 compressed | `NVLZ` | liblzma `lzma_raw_decoder` |

## Environment Variables

| Variable | Effect |
|----------|--------|
| `CUDA_VISIBLE_DEVICES` | Restricts which GPUs are visible. `CUDA_VISIBLE_DEVICES=0` forces single GPU 0. |
